Introduction


Welcome to the LTEviz application. This manual provides a complete guide to using the features of this tool for exploring temporal variant data obtained through next generation sequencing technologies.

Our application is primarily designed to explore and visualize HIV-1 variants from the Long-Term Evolution Experiment (LTEE) published in (link to DOI).

Alternatively, it allows users to upload their own datasets using standard format files to:

  • explore the variant data through an interactive data table
  • visualize variant data through an integrated genome browser over multiple lineages
  • track variant trajectories over time

The application is divided into several sections, each accessible from the navigation bar at the top of the page. First step would be to navigate to the Data Upload tab to load the data.



Data Upload


In this section, you can either load the pre-existing LTEE HIV-1 dataset or upload your own experimental dataset for visualization and analysis.


➡️ Sidebar Controls

A radio button allows you to select the source of genomic data:

  • LTEE HIV-1 (default)
  • Own data

Option 1: Load LTEE HIV-1 Dataset

Click the Load LTEE HIV-1 data button to automatically load a preprocessed version of the HIV-1 NL4_3 genome along with its annotation and variant data.

Option 2: Upload Own Data

Switch the Data Source to Own data to manually upload your files by providing absolute file paths.

Required Inputs:

  • Genome name
    An arbitrary name for your reference genome (e.g., "HIV-1 NL4-3").

  • Reference genome
    A FASTA file containing your reference genome sequence.

  • Annotation file
    A GFF3 file containing the genomic feature annotations. Such files for the model organism of interest can. be downloaded from the NCBI or Ensembl databases.

  • Variant file
    A VCF file containing the variant calls. This file should be in VCF format, which is a standard format for storing variant data. For specifications on the INFO field of the VCF file, please see below.


INFO Field Specifications

The INFO field of the VCF file can be used to store metadata information regarding sequencing quality, functional annotation, and temporal information. The following tags are required:

  • COVERAGE
    Sequencing coverage at the variant position.
  • AF
    Allele frequency of the variant in the population.
  • PASSAGE
    Passage number indicating the time point of the variant in the experiment.
  • LINE
    Lineage identifier for the variant, e.g., “L1”, “L2”, etc.

Additional optional tags can be added for a more completed annotation of the variants:

  • MT
    Mutation type, e.g., “P” for point variant, “I” for small insertion, and “D” for deletion.


📝 Notes and Tips

  • The FASTA reference file must be accompanied by an index file (.fai) in the same directory. The index file can be generated by samtools faidx command.
  • The VCF variant file must be accompanied by an index file (.csi) in the same directory. The index file can be generated by bcftools index command.



Data Table


The Variant Table tab allows you to explore, filter, and inspect annotated variant data. The table updates in real time as you apply filters from the sidebar.


➡️ Sidebar Controls

The left panel offers several input controls for filtering the variant dataset:

Lineage Selection

Control: Select lineage(s)

Choose one or more experimental lineages to include in the table. Each lineage represents a sample or replicate (e.g., MT-2_1, MT-4_2). You can select multiple lineages simultaneously.

Passage Selection

Control: Passage

Select the time point (passage number) of interest. Available passages range from 10 to 500 (every 10 passages). Only variants present at the selected passage will be shown.

Allele Frequency Range

Control: Allele Frequency Range

Use this slider to filter variants based on their allele frequency (AF). This helps focus on variants with higher or lower prevalence. The range is from 0.00 to 1.00, and the default is 0.01 to 1.00.

Translational Impact

Control: Translational Impact(s)

Choose one or more categories of functional impact on protein translation:

  • Any: No filtering by impact (default)
  • Untranslated (U): Located in UTR or intergenic regions
  • Synonymous (S): No change in amino acid
  • Non-synonymous (N): Alters amino acid sequence

Selecting “Any” automatically includes all categories.

Filter Mode

Control: Filter by: Position / Feature

This radio button lets you choose how to further refine the variant list:

  • Position: Filter by specific genomic position or range
  • Feature: Filter by gene or feature (e.g., gag, pol, etc.)

Position Input

Shown only when “Position” is selected above.

Control: Position

You can enter: - A single position (e.g., 9512) - A range (e.g., 9510-9522) - Whitespace is ignored

Invalid input triggers a helpful error message.

Feature Selection

Shown only when “Feature” is selected above.

Control: Feature(s)

Check one or more genomic features to filter variants. Selecting “All” includes all features automatically.


𓊂 Main Panel: Variant Table

The main panel displays a dynamic table of variants based on the filters applied. Columns in the table include (depending on your data):

  • genomic_pos: Genomic position of the variant
  • allele_freq: Allele frequency at selected passage
  • exp_line: Experimental lineage
  • feature: Associated gene or region
  • effect_simplified: Translational impact (A/U/S/N)
  • And possibly others from the dataset

Features of the table:

  • Pagination (10 rows per page)
  • Search box (top-right)
  • Sortable columns (click headers)
  • Responsive updates to filtering


📝 Notes and Tips

  • To view variants across all lineages at a passage, select all lineages.
  • Use Allele Frequency to highlight highly prevalent mutations.
  • Toggle between Position and Feature for flexible region-specific filtering.
  • Invalid inputs (e.g., abc or reversed ranges) show notifications without crashing the app.



Genome Browser


The Genome Browser tab integrates the IGV.js genome visualization tool via the igvShiny package. This view allows users to explore the genome of the model organism of interest, overlay annotated features, and dynamically load variant tracks filtered by lineage, passage, and allele frequency.


➡️ Sidebar Controls

The left sidebar includes tools for loading and managing tracks:

Add Genomic Features

Button: Add Genomic Features

This loads annotated features (from a GFF3 file) onto the IGV view. Features may include: - Genes (gag, pol, etc.) - Repeat regions - Regulatory elements

The annotation track is shown in expanded mode with color-coded feature types for easier distinction.

Add Variant Track

Button: Add Variant Track

Clicking this adds a new variant filtering panel and prepares to overlay a new track.

Each variant track allows fine-grained filtering with the following controls:

Lineage

Dropdown: Lineage

Select the experimental lineage (e.g., MT-2_1, MT-4_2) for which you want to visualize variants.

Passage

Dropdown: Passage

Choose the passage (generation time point) from 10 to 500. Only variants detected at this passage will be shown.

Allele Frequency

Slider: Allele Frequency Range

Specify the allele frequency (AF) range to include. You can visualize variants from low-frequency emergents to high-frequency fixations. Default range is 0.01 to 1.00.

Load Variants

Button: Load Variants

After setting filters, click this to load the corresponding variants as a new track in the IGV browser. If no variants are found, a warning will appear.

🔁 You can repeat this process to add multiple tracks for comparison between conditions or time points.

Remove User Tracks

Button: Remove User Tracks

Clears all added variant tracks and resets the sidebar UI to its initial state.


🧬 Main Panel: IGV Browser

The right-hand panel contains the interactive IGV genome browser initialized with:

  • Genome: HIV-1 NL4-3
  • Default locus: AF324493.2:600–700
  • Reference FASTA and index files loaded locally
  • GFF3 annotations available via Add Genomic Features

Key Features:

  • Zoomable and scrollable genomic region
  • Drag-to-pan functionality
  • Tooltips on feature hover
  • Dynamic loading of tracks
  • Supports multiple overlaid tracks
  • Color-coded feature types and variant tracks


🛠 Troubleshooting

  • “No variants found for selected filters”: Your filters may be too strict. Try broadening the AF range or selecting a different passage.
  • Track not appearing: Ensure that the correct filters are set and that genomic features have been added if needed.
  • Performance issues: Avoid loading too many tracks at once if browser becomes sluggish.


📝 Notes & Tips

  • All files are accessed locally (FASTA, GFF3, and VCF).
  • Variant metadata includes AF, LINE, and PASSAGE, which are used for filtering.
  • Variants are color-encoded per track but not per consequence or strand.



Frequency Tracker


The Frequency Tracker tab provides an interactive, lineage-specific visualization of how individual variants change in frequency over experimental passages. This is especially useful for understanding mutation dynamics, selection patterns, and lineage divergence in long-term evolution experiments.


➡️ Sidebar Controls

The left panel contains two main inputs that determine what will be plotted:

Select Lineage(s)

Input Type: checkboxGroupInput
Label: Select lineage(s):

Choose one or more lineages to include in the plot. Each lineage corresponds to a replicate or condition from your experiment (e.g., MT-2_1, MT-4_2).

Insert Variant ID

Input Type: textInput
Label: Insert variant id:
Format Required: position_ref_alt (e.g., 566_G_A)

Specify the mutation you want to track. The format must follow the pattern:

For example: - 566_G_A means a G-to-A mutation at position 566. - 1234_T_C means a T-to-C mutation at position 1234.


📈 Main Panel: Variant Trajectory Plot

The right-hand panel shows the trajectory of the selected mutation(s) over time (measured in passages).

Plot Details:

  • X-axis: Passage number (transfer/time point)
  • Y-axis: Variant frequency (on a log scale)
    • Displayed as log10-scaled but labeled for interpretability:
      • 1 → 100%
      • 0.1 → 10%
      • 0.01 → 1%
      • 0.001 → 0.1%
  • Line color: Indicates experimental lineage
  • Interactive tooltips (via Plotly): Hover to inspect passage and frequency values


🛠 Troubleshooting

  • Blank plot?
    • Double-check the mutation ID format (e.g., no extra spaces).
    • Ensure at least one lineage is selected.
    • Confirm the mutation exists in your data.
  • Y-axis hard to interpret?
    • Note that the plot uses log10(frequency) to capture small values clearly.
  • Multiple lines overlap?
    • You can deselect lineages to declutter the plot.


📝 Notes & Tips

  • You can compare the same mutation across multiple lineages.
  • Only mutations present in your dataset will result in plotted lines. Typos or non-existent variants will yield an empty plot.
  • Backend data: Uses the variant_data data frame filtered by mut_info and exp_line.
  • Each curve is uniquely identified by mut_info + lineage (to allow multiple lineages per variant).
  • Styling: The plot uses a custom publication-style ggplot theme with large, readable text.

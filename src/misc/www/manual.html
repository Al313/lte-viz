Introduction


Welcome to the LTEEviz application. This manual provides a complete guide to using the features of this tool for exploring temporal mutation data obtained through next generation sequencing technologies.

LTEEviz is designed to access, explore, and visualize HIV-1 mutations from the Long-Term Evolution Experiment (LTEE) published in (link to DOI).

The application is divided into several sections, each accessible from the navigation box at the top of the page. Below you can find a brief overview of each section:



Data Access


In this section, you can download the processed data for LTEE HIV-1 dataset.


➡️ Sidebar Controls


Dataset Selection

First radio button allows you to select the source of the genomic data for download:

  • LTEE HIV-1 EXP-2E (default)
  • LTEE HIV-1 EXP-SB (currecntly unavailable)

Data Format Selection

Second radio button allows you to select the format of the genomic data for download:

  • VCF (default)
  • CSV



Mutation Table


The mutation Table tab allows you to explore, filter, and inspect annotated mutation data. The table updates in real time as you apply filters from the sidebar.

This view is intended for detailed inspection of mutation-level data at a specific passage, lineage, and frequency range.

➡️ Sidebar Controls

The left-hand panel provides several controls to filter the mutation dataset:

Lineage Selection

  • Input: Checkbox group
  • Label: Select lineage(s):

Choose one or more experimental lineages to include in the table. You can select multiple lines simultaneously.

Passage Selection

  • Input: Dropdown
  • Label: Passage:

Select the passage (time point) to display mutations from. Available passages range from 10 to 500, in steps of 10.
Only mutations detected at the selected passage will be shown.

Variant Frequency Range

  • Input: Slider
  • Label: Variant Frequency Range:

Filter mutations based on their allele (variant) frequency.

  • Minimum: 0
  • Maximum: 1
  • Default range: 0.01 to 1.00
  • Step size: 0.01

Filter Mode

  • Input: Radio buttons
  • Label: Filter by:

Choose how mutations should be further refined:

  • Feature (default): Filter mutations by genomic feature and translational impact
  • Position: Filter mutations by exact genomic position or position range

Only the controls relevant to the selected mode are displayed.

1. Feature-based Filtering

Displayed when Filter by = Feature is selected.

Translational Impact(s)

  • Input: Multi-select dropdown
  • Label: Translational Impact(s):

Select one or more categories describing the functional impact of mutations:

  • Select All
  • Untranslated
  • Synonymous
  • Non-synonymous

Selecting Select All automatically expands the selection to include:

  • Untranslated
  • Synonymous
  • Non-synonymous

Feature(s)

  • Input: Checkbox group
  • Label: Feature(s):

Select one or more genomic features (genes or regions) to include.

All / None Toggle

  • Input: Checkbox
  • Label: All/None

When checked (default), all features are automatically selected.
Unchecking clears the selection, allowing manual feature choice.

2. Position-based Filtering

Displayed when Filter by = Position is selected.

Position

  • Input: Text input
  • Label: Position:

Specify a genomic position or range:

  • Single position: 678
  • Position range: 678-700

Whitespace is ignored.

Input validation:

  • If the start position is greater than the end position, an error notification is shown.
  • Invalid formats (e.g. letters or malformed ranges) trigger a helpful error message.
  • Invalid input results in an empty table rather than crashing the app.


𓊂 Main Panel: Mutation Table

The main panel displays a dynamic table of mutations based on the filters applied. The table displays the following fields:

  • genomic_pos: Genomic position of the mutation
  • variant_freq: Variant frequency at selected passage
  • exp_line: Experimental lineage
  • feature: Associated gene or region
  • effect_simplified: Translational impact (A/U/S/N)
  • Additional annotation fields present in the dataset

Table Features:

  • Pagination (10 rows per page)
  • Search box (top-right)
  • Sortable columns (click headers)
  • Responsive updates to filtering


📝 Notes and Tips

  • To view mutations across all lineages at a passage, select all lineages.
  • Use Variant Frequency to filter for highly prevalent mutations.
  • Toggle between Position and Feature for flexible region-specific filtering.
  • Invalid inputs (e.g., abc or reversed ranges) invoke a warning message.



Genome Browser


The Genome Browser tab integrates the IGV.js genome visualization tool via the igvShiny package. This view allows users to explore the genome of the model organism of interest, overlay annotated features, and dynamically load mutation tracks filtered by lineage, passage, and variant frequency.


➡️ Sidebar Controls

The left sidebar includes tools for loading and managing tracks:

Add Genomic Features

Button: Add Genomic Features

This loads annotated features (from a GFF3 file) onto the IGV view. Features may include: - Genes (gag, pol, etc.) - Repeat regions - Regulatory elements

The annotation track is shown in expanded mode with color-coded feature types for easier distinction.

Add Mutation Track

Button: Add Mutation Track

Clicking this adds a new mutation filtering panel and prepares to overlay a new track. You can add up to 4 mutation filtering panels and corresponding tracks simultaneously.

Each mutation track allows fine-grained filtering with the following controls:

Lineage

Dropdown: lineage

Select the experimental lineage (e.g., MT-2_1, MT-4_2) for which you want to visualize mutations.

Passage

Dropdown: Passage

Choose the passage (generation time point) from 10 to 500. Only mutations detected at this passage will be shown.

Variant Frequency

Slider: Variant Frequency Range

Specify the variant frequency (VF) range to include. You can visualize mutations from low-frequency emergents to high-frequency fixations. Default range is 0.01 to 1.00.

Load Mutations

Button: Load mutations

After setting filters, click this to load the corresponding mutations as a new track in the IGV browser. If no mutations are found, a warning will appear.

🔁 You can repeat this process to add multiple tracks for comparison between conditions or time points.

Remove User Tracks

Button: Remove User Tracks

Clears all added mutation tracks and resets the sidebar UI to its initial state.


🧬 Main Panel: IGV Browser

The right-hand panel contains the interactive IGV genome browser initialized with:

  • Genome: HIV-1 NL4-3
  • Default locus: AF324493.2:600–700
  • GFF3 annotations available via Add Genomic Features

Key Features:

  • Zoomable and scrollable genomic region
  • Drag-to-pan functionality
  • Fetching detailed information on mutations by clicking on them
  • Tooltips on feature hover
  • Dynamic loading of tracks
  • Supports multiple overlaid tracks
  • Color-coded feature types and mutation tracks


🛠 Troubleshooting

  • “No mutations found for selected filters”: Your filters may be too strict. Try broadening the VF range or selecting a different passage.
  • Track not appearing: Ensure that the correct filters are set and that genomic features have been added if needed.
  • Performance issues: Avoid loading too many tracks at once if browser becomes sluggish.


📝 Notes & Tips

  • Hover your pointer over each mutation to display its metadata. Mutation metadata includes VF, LINE, and PASSAGE, which are used for filtering.



Mutation Frequency Dynamics


The Frequency Dynamics tab provides interactive visualizations of mutation frequency behavior across experimental lineages. It supports two complementary views: (i) an animated frequency distribution summarizing genome-wide dynamics, and (ii) frequency trajectories of specific mutations over time.


➡️ Sidebar Controls

The left panel contains inputs that control what is displayed in the main panel.

Display Mode

Input Type: radioButtons
Label: Display mode:

Choose between two visualization modes:

  • Frequency distribution – displays an animated GIF summarizing mutation frequency distributions over time.
  • Frequency trajectory – enables detailed tracking of one or more specific mutations across passages.

Additional controls appear only when Frequency trajectory is selected:

Select Lineage(s)

Input Type: checkboxGroupInput Label: Select lineage(s):

Choose one or more experimental lineages to include in the plot (e.g., MT-2_1, MT-4_2).
Each lineage is plotted using a distinct color.

Insert Mutation ID(s)

Input Type: textInput (dynamic, up to 4)
Labels:
- Insert mutation id 1:
- Insert mutation id 2 (optional):
- Insert mutation id 3 (optional):
- Insert mutation id 4 (optional):

Format Required: position_ref_alt (e.g., 566_G_A)

Specify one or more mutations to track. Mutation IDs must follow the format:

  • 566_G_A → G-to-A mutation at position 566
  • 1234_T_C → T-to-C mutation at position 1234

Use the “Add another mutation” button to include additional mutations (maximum of four).

Load Plot

Input Type: actionButton
Label: Load Plot

After selecting lineages and entering mutation IDs, click Load Plot to render the trajectory plot.
The plot is not updated automatically to avoid unnecessary recomputation.


📈 Main Panel: Frequency Distribution and Dynamics

When Frequency distribution is selected, the main panel displays a pre-generated animated GIF illustrating the distribution of mutation frequencies over time.

This view provides a temporal overview of genome-wide frequency dynamics across passages.


When Frequency trajectory is selected, the right-hand panel displays the temporal trajectories of the selected mutation(s) across passages. Details of the plot:

  • X-axis: Transfer number (passage)
  • Y-axis: Variant frequency (log₁₀ scale)
    • Common reference points:
      • 1 → 100%
      • 0.1 → 10%
      • 0.01 → 1%
  • Line color: Experimental lineage
  • Line type: Mutation ID
    (allows multiple mutations to be distinguished within the same lineage)
  • Grouping: Each curve represents a unique mutation × lineage combination
  • Interactive tooltips (via Plotly): Hover to inspect passage number and frequency

The legend is displayed horizontally above the plot for compact comparison.


🛠 Troubleshooting

  • No lines appear after clicking “Load Plot”?
    • Check that mutation IDs follow the correct format (position_ref_alt).
    • Ensure at least one lineage is selected.
    • Confirm that the specified mutation(s) exist in the dataset.
  • Received a “No records found” message?
    • The selected combination of mutation ID(s) and lineage(s) does not occur in the data.
  • Y-axis difficult to interpret?
    • Frequencies are shown on a log₁₀ scale to clearly display low-frequency variants.
  • Plot looks cluttered?
    • Reduce the number of selected lineages or mutations.


📝 Notes & Tips

  • You can compare multiple mutations simultaneously, across one or more lineages.
  • A maximum of four mutation IDs can be visualized at once.
  • Only mutations present in the dataset will be plotted; invalid or misspelled IDs will produce no output.
  • Backend data: Uses the mutation_data data frame filtered by mut_info and exp_line.
  • Each trajectory is uniquely identified by mut_info + lineage.
  • Styling: The plot uses a clean, publication-oriented ggplot theme optimized for readability.

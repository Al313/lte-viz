Introduction


Welcome to the LTEEviz application. This manual provides a complete guide to using the features of this tool for exploring temporal mutation data obtained through next generation sequencing technologies.

LTEEviz is designed to access, explore, and visualize HIV-1 mutations from the Long-Term Evolution Experiment (LTEE) published in (link to DOI).

The application is divided into several sections, each accessible from the navigation box at the top of the page. Below you can find a brief overview of each section:



Data Access


In this section, you can download the processed data for LTEE HIV-1 dataset.


➡️ Sidebar Controls


Dataset Selection

First radio button allows you to select the source of the genomic data for download:

  • LTEE HIV-1 EXP-2E (default)
  • LTEE HIV-1 EXP-SB

Data Format Selection

Second radio button allows you to select the format of the genomic data for download:

  • VCF (default)
  • CSV



Mutation Table


The mutation Table tab allows you to explore, filter, and inspect annotated mutation data. The table updates in real time as you apply filters from the sidebar.


➡️ Sidebar Controls

The left panel offers several input controls for filtering the mutation dataset:

Lineage Selection

Choose one or more experimental lineages to include in the table. Each lineage represents a sample or replicate (e.g., MT-2_1, MT-4_2). You can select multiple lineages simultaneously.

Passage Selection

Select the time point (passage number) of interest. Available passages range from 10 to 500 (every 10 passages). Only mutations present at the selected passage will be shown.

Variant Frequency Range

Use this slider to filter mutations based on their variant frequency (VF). This helps focus on mutations with higher or lower prevalence. The range is from 0.00 to 1.00, and the default is 0.01 to 1.00.

Translational Impact

Choose one or more categories of functional impact on protein translation:

  • Any: No filtering by impact (default)
  • Untranslated (U): Located in UTR or intergenic regions
  • Synonymous (S): No change in amino acid
  • Non-synonymous (N): Alters amino acid sequence

Selecting “Any” automatically includes all categories.

Filter Mode

This radio button lets you choose how to further refine the mutation list:

  • Position: Filter by specific genomic position or range
  • Feature: Filter by gene or feature (e.g., gag, pol, etc.)

Position Input

Shown only when “Position” is selected above.

You can enter: - A single position (e.g., 9512) - A range (e.g., 9510-9522) - Whitespace is ignored

Invalid input triggers a helpful error message.

Feature Selection

Shown only when “Feature” is selected above.

Check one or more genomic features to filter mutations. Selecting “All” includes all features automatically.


𓊂 Main Panel: Mutation Table

The main panel displays a dynamic table of mutations based on the filters applied. Columns in the table include (depending on your data):

  • genomic_pos: Genomic position of the mutation
  • variant_freq: Variant frequency at selected passage
  • exp_line: Experimental lineage
  • feature: Associated gene or region
  • effect_simplified: Translational impact (A/U/S/N)
  • And possibly others from the dataset

Features of the table:

  • Pagination (10 rows per page)
  • Search box (top-right)
  • Sortable columns (click headers)
  • Responsive updates to filtering


📝 Notes and Tips

  • To view mutations across all lineages at a passage, select all lineages.
  • Use Variant Frequency to highlight highly prevalent mutations.
  • Toggle between Position and Feature for flexible region-specific filtering.
  • Invalid inputs (e.g., abc or reversed ranges) show notifications without crashing the app.



Genome Browser


The Genome Browser tab integrates the IGV.js genome visualization tool via the igvShiny package. This view allows users to explore the genome of the model organism of interest, overlay annotated features, and dynamically load mutation tracks filtered by lineage, passage, and variant frequency.


➡️ Sidebar Controls

The left sidebar includes tools for loading and managing tracks:

Add Genomic Features

Button: Add Genomic Features

This loads annotated features (from a GFF3 file) onto the IGV view. Features may include: - Genes (gag, pol, etc.) - Repeat regions - Regulatory elements

The annotation track is shown in expanded mode with color-coded feature types for easier distinction.

Add Mutation Track

Button: Add Mutation Track

Clicking this adds a new mutation filtering panel and prepares to overlay a new track.

Each mutation track allows fine-grained filtering with the following controls:

Lineage

Dropdown: Lineage

Select the experimental lineage (e.g., MT-2_1, MT-4_2) for which you want to visualize mutations.

Passage

Dropdown: Passage

Choose the passage (generation time point) from 10 to 500. Only mutations detected at this passage will be shown.

Variant Frequency

Slider: Variant Frequency Range

Specify the variant frequency (VF) range to include. You can visualize mutations from low-frequency emergents to high-frequency fixations. Default range is 0.01 to 1.00.

Load Mutations

Button: Load mutations

After setting filters, click this to load the corresponding mutations as a new track in the IGV browser. If no mutations are found, a warning will appear.

🔁 You can repeat this process to add multiple tracks for comparison between conditions or time points.

Remove User Tracks

Button: Remove User Tracks

Clears all added mutation tracks and resets the sidebar UI to its initial state.


🧬 Main Panel: IGV Browser

The right-hand panel contains the interactive IGV genome browser initialized with:

  • Genome: HIV-1 NL4-3
  • Default locus: AF324493.2:600–700
  • Reference FASTA and index files loaded locally
  • GFF3 annotations available via Add Genomic Features

Key Features:

  • Zoomable and scrollable genomic region
  • Drag-to-pan functionality
  • Fetching detailed information on mutations by clicking on them
  • Tooltips on feature hover
  • Dynamic loading of tracks
  • Supports multiple overlaid tracks
  • Color-coded feature types and mutation tracks


🛠 Troubleshooting

  • “No mutations found for selected filters”: Your filters may be too strict. Try broadening the VF range or selecting a different passage.
  • Track not appearing: Ensure that the correct filters are set and that genomic features have been added if needed.
  • Performance issues: Avoid loading too many tracks at once if browser becomes sluggish.


📝 Notes & Tips

  • All files are accessed locally (FASTA, GFF3, and VCF).
  • Mutation metadata includes VF, LINE, and PASSAGE, which are used for filtering.
  • Mutations are color-encoded per track but not per consequence or strand.



Mutation Frequency Dynamics


The Frequency Tracker tab provides an interactive, lineage-specific visualization of how individual mutations change in frequency over experimental passages. This is especially useful for understanding mutation dynamics, selection patterns, and lineage divergence in long-term evolution experiments.


➡️ Sidebar Controls

The left panel contains two main inputs that determine what will be plotted:

Select Lineage(s)

Input Type: checkboxGroupInput
Label: Select lineage(s):

Choose one or more lineages to include in the plot. Each lineage corresponds to a replicate or condition from your experiment (e.g., MT-2_1, MT-4_2).

Insert Mutation ID

Input Type: textInput
Label: Insert mutation id:
Format Required: position_ref_alt (e.g., 566_G_A)

Specify the mutation you want to track. The format must follow the pattern:

For example: - 566_G_A means a G-to-A mutation at position 566. - 1234_T_C means a T-to-C mutation at position 1234.


📈 Main Panel: Mutation Trajectory Plot

The right-hand panel shows the trajectory of the selected mutation(s) over time (measured in passages).

Plot Details:

  • X-axis: Passage number (transfer/time point)
  • Y-axis: Mutation frequency (on a log scale)
    • Displayed as log10-scaled but labeled for interpretability:
      • 1 → 100%
      • 0.1 → 10%
      • 0.01 → 1%
      • 0.001 → 0.1%
  • Line color: Indicates experimental lineage
  • Interactive tooltips (via Plotly): Hover to inspect passage and frequency values


🛠 Troubleshooting

  • Blank plot?
    • Double-check the mutation ID format (e.g., no extra spaces).
    • Ensure at least one lineage is selected.
    • Confirm the mutation exists in your data.
  • Y-axis hard to interpret?
    • Note that the plot uses log10(frequency) to capture small values clearly.
  • Multiple lines overlap?
    • You can deselect lineages to declutter the plot.


📝 Notes & Tips

  • You can compare the same mutation across multiple lineages.
  • Only mutations present in your dataset will result in plotted lines. Typos or non-existent mutations will yield an empty plot.
  • Backend data: Uses the mutation_data data frame filtered by mut_info and exp_line.
  • Each curve is uniquely identified by mut_info + lineage (to allow multiple lineages per mutation).
  • Styling: The plot uses a custom publication-style ggplot theme with large, readable text.
